# Zam 1.4 CSV Import/Export Hardening

## Summary

Zam 1.4 is a release-hardening pass for the CSV import/export foundation.

This is not a new feature release. The goal is to prove that Zam can import transaction data, export transaction data, and re-import Zam's own export without data-shape surprises.

## Current Status

Started: June 18, 2026

Completed in the first 1.4 slice:

- Updated CSV import/export tests for the approved non-blocking import-complete notification.
- Verified import feedback is optional and no longer forced after import.
- Verified focused CSV import/export Playwright tests pass.
- Added import/export/privacy/release docs under `docs/`.

## Primary QA Commands

```powershell
npx playwright test tests/csv-import-review.spec.js tests/csv-export-foundation.spec.js
npm run build
```

## Final Acceptance Criteria

- Basic CSV import passes.
- Hard CSV import passes.
- Stress CSV import displays all rows.
- Duplicate rows are detected.
- Invalid rows are blocked.
- Needs Review rows are surfaced.
- Strict invalid dates are rejected.
- Debit/Credit columns satisfy the Amount requirement.
- Single Amount column still works.
- Imported transaction metadata remains visible after edits.
- Manual transactions do not show Import Details.
- Current Month export works.
- Last Month export works.
- Custom Range export works.
- All Time export works.
- Account filter works.
- Category filter works.
- Empty export filter state works.
- Export filename ends in `.csv`.
- Export includes the header row.
- Dates export as `YYYY-MM-DD`.
- Amounts export as plain signed decimals.
- Export shows plaintext warning before download.
- Zam export can be re-imported by Zam.
- Roundtrip preserves dates, descriptions, amounts, categories, accounts/payment methods, and notes.

## Privacy / Legal Review

Expected outcome:

Legal/marketing checked: no policy change needed.

Reason:

- Import preview remains local.
- Export remains local.
- Raw CSV data is not uploaded for parsing.
- Exported files are downloaded directly by the user.
- Support feedback does not include transaction values by default.

Relevant docs:

- `docs/CSV_IMPORT.md`
- `docs/CSV_EXPORT.md`

- `documentation/Major-Change-Release-Gate.md`

## Out of Scope

Do not add these in v1.4:

- Reconciliation
- Merchant cleanup
- Transfer detection
- Statement balancing
- PDF reports
- Google Drive import/export
- Dropbox import/export
- Multi-file import
- Bulk edit
- Bulk categorize
- Saved export templates
- Custom export columns
- AI categorization

## Known Design Debt

- Diagnostic/import review views can be visually dense for large files.
- Raw CSV view is intentionally utilitarian for QA and audit purposes.
- Further mobile polish can wait unless a layout blocks import/export completion.
