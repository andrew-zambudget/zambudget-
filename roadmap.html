Product Roadmap

What is live, what is being tested, and what only happens if Zam! proves useful enough to continue.

Beta direction: Zam! is being tested in public around reliability, data safety, billing, and Cloud Sync. Continued development depends on real user feedback, whether the app is useful enough to keep entering budgets, and whether Premium support can at least cover infrastructure and continued development time.
Live Beta

Core Product

The working budget app: local-first budgeting, optional signed-in cloud protection, and the core workflows needed to enter and review a real budget.

  • 💰
    Income & Expense Tracking
    Live
  • 📂
    Categories with edit mode, drag sorting, up/down controls, and position entry
    Live
  • 🎛️
    Income, Savings, Debt, Add, Recent, and Calendar workflows
    Live
  • 🛡️
    Privacy-first local budget storage with no trackers or ads
    Live
  • 🔒
    Google OAuth and email magic-link sign-in
    Live; Apple sign-in paused until demand justifies it
  • 🌙

Active Beta

Reliability, Data Safety, and Conversion

This is the current focus. The app needs to be trustworthy before it gets more feature work.

  • ☁️
    Cloud Sync encrypted sync, version history, and sign-in recovery
    Live beta; still being stress-tested
  • 🔐
    Graceful sign-out, rough logout, hard refresh, and fresh-session recovery testing
    Active beta QA
  • 📊
    Premium billing and Stripe upgrade path
    Live beta path; keep testing before wider promotion
  • 📱
    GitLab and Cloudflare deployment workflow
    Live; auto-deploys from pushed main branch
  • 📥
    Beta user feedback: is it useful, stable, and worth paying to keep alive?
    Decision input for continued development
