Data Ownership

You own all the data you enter. If you sign in, Cloud Sync protects your budget by default with an encrypted cloud vault that Zam! cannot decrypt without your recovery key. If you use Zam! without signing in, your budget remains browser-only.

  • Browser-only data can be removed by clearing browser or site data, and Zam! cannot recover it after local storage is lost
  • Cloud Sync data is encrypted on your device before upload and remains unreadable to Zam!
  • A trusted browser may store a non-extractable WebCrypto key in IndexedDB so Cloud Sync can keep syncing after refresh. This local browser key is not uploaded to Zam!, is not part of Account > Devices records, cannot be displayed as the recovery-key text, and is not a substitute for saving your recovery key
  • Free Tier Cloud Sync capacity is managed with privacy-minimal active browser leases. Account > Devices may link a known browser access record to an existing opaque sync slot hash so you can see and release unmatched slots. If a browser does not check in for 60 minutes, the slot can be released automatically. This link is not a new device identifier and does not include device names, user agents, IP-derived locations, readable budget data, or recovery keys
